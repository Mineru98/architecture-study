# GraphQL 패턴

## 개념
GraphQL은 클라이언트가 필요한 데이터 형태를 선언적으로 요청하는 방식이다.
- 단일 엔드포인트 중심
- 쿼리/뮤테이션/구독 분리
- 타입 시스템 기반 계약

## 언제 쓰는지
- 화면별 필요한 필드가 달라 과다 응답/과소 응답 문제가 자주 생길 때
- 여러 도메인을 한 번의 요청으로 조합해야 할 때
- 모바일 대역폭이나 네트워크 비효율이 문제일 때

## 기본 데이터 흐름
1. React가 쿼리 문서를 서버에 전송
2. GraphQL Resolver가 타입별 필드 해석
3. 서비스 계층에서 하위 리졸버 호출
4. 응답을 필요한 필드만 조립해 반환

## NestJS + React 예시 구상
- 시나리오: 학습 대시보드에서 학습자/과제/진행률을 한 번에 조회
- NestJS: GraphQL module + Code First/Schema First로 타입 정의
- React: `@apollo/client` 또는 유사 클라이언트로 쿼리 문서 작성
- 화면별 필요한 최소 필드만 요청하도록 View 의존 최소화

## 스터디 범위
- 스키마/타입 계약 설계
- Resolver에서 N+1 방지 전략
- 캐시 정책과 부분 갱신
- 소규모 화면 2개 이상에서 동일 쿼리 구조 재사용 실습
