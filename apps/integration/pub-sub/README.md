# Pub/Sub 패턴

## 개념
Pub/Sub은 발행자와 구독자의 직접 결합을 없애 메시지를 브로커 중심으로 분배하는 패턴이다.
- 메시지 발행은 이벤트 의미 중심
- 구독자는 타입/토픽 기반으로 수신

## 언제 쓰는지
- 다수의 하위 처리기가 하나의 이벤트를 받아야 할 때
- 이벤트 기반 아키텍처에서 결합도를 낮추고 싶을 때
- 기능 추가 시 기존 발행 코드를 변경하지 않기 원하는 경우

## 기본 데이터 흐름
1. Publisher가 이벤트 발행
2. Broker가 구독 규칙에 매칭되는 소비자에게 전달
3. 각 구독자 이벤트를 독립적으로 처리
4. 필요시 재시도/Dead Letter로 내결함성 확보

## NestJS + React 예시 구상
- 시나리오: 학습 과제 완료 시 알림/통계/포인트 부여 이벤트 분기
- NestJS: 완료 이벤트 토픽을 emit 후 Analytics/Notification/Badge 모듈이 구독
- React: 사용자는 알림 토스트와 배지 갱신을 실시간 수신
- 중복 구독 처리를 idempotent 키로 정리

## 스터디 범위
- 이벤트 스키마 표준화
- 구독자별 실패 정책 분리
- 테스트에서 순차/병렬 구독 시나리오 검증
- 한 이벤트가 여러 도메인에 미치는 영향 분석
