# RPC 패턴

## 개념
RPC는 원격 프로시저를 마치 로컬 함수 호출처럼 다루는 통신 방식이다.
- 메서드/파라미터 중심 인터페이스
- 서비스 간 강한 계약 및 직렬화 규약 필요

## 언제 쓰는지
- 내부 마이크로서비스 간 호출 성능/명세 일치가 중요할 때
- 복잡한 오퍼레이션을 하나의 원자적 호출로 묶고 싶을 때
- REST보다 낮은 오버헤드/명확한 메서드 호출이 필요할 때

## 기본 데이터 흐름
1. Client가 특정 메서드/메시지 스키마로 요청
2. 서버가 직렬화된 요청 파싱
3. 핸들러에서 도메인 실행 후 응답 직렬화
4. 네트워크 계층을 통해 결과 반환

## NestJS + React 예시 구상
- 시나리오: 학습 세션 시작/종료를 하나의 호출로 제어
- NestJS: gRPC Controller 기반 서비스(또는 HTTP/RPC 적응 계층) 구성
- React: 버튼 클릭 시 세션 제어 RPC 엔드포인트 호출
- 요청 실패시 타임아웃/재시도 정책을 공통화

## 스터디 범위
- 인터페이스 정의와 버전 관리
- 에러 코드/타임아웃/재시도 정책
- 클라이언트-서버 계약 계약 테스트
- 동작을 동기/비동기로 구분해 문서화
