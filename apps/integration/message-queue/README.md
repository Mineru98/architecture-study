# Message Queue 패턴

## 개념
Message Queue는 메시지를 큐에 적재하고 소비자가 처리 완료 후 다음 단계로 넘기는 비동기 통합 패턴이다.
- 비동기 처리 및 백프레셔 완화
- 소비자 확장성 확보

## 언제 쓰는지
- 즉시 응답이 필요 없는 작업(리포트 생성, 메일 발송, 이미지 처리)에 적합
- 트래픽 피크를 흡수해야 할 때
- 처리 순서 보장이나 재시도 정책이 필요한 백엔드 파이프라인에서 사용

## 기본 데이터 흐름
1. API가 작업 요청을 큐에 적재
2. Consumer가 큐에서 메시지 polling/푸시 수신
3. 처리 후 결과 상태 갱신
4. 클라이언트는 상태 조회 또는 알림으로 완료 확인

## NestJS + React 예시 구상
- 시나리오: 학습자료 PDF 생성 요청
- NestJS: 제출 시 작업 메시지를 큐에 넣고 즉시 요청 ID 반환
- Worker가 생성 완료 후 상태 변경 이벤트 발행
- React: 작업 상태를 poll/WebSocket으로 확인하고 완료 링크 표시

## 스터디 범위
- 큐 드라이버 선택(메모리/Redis/RabbitMQ)과 장단점
- 데드레터/재시도 정책 설계
- 멱등성 키와 중복 처리 방지
- 1건 처리 실패가 전체 파이프라인에 미치는 영향 분석
