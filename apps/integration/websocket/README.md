# WebSocket 패턴

## 개념
WebSocket은 장기 연결을 통해 서버가 클라이언트에 실시간으로 이벤트를 푸시하는 통신 방식이다.
- 양방향 통신
- 연결 유지 기반 이벤트 스트림

## 언제 쓰는지
- 실시간 알림, 채팅, 협업 커서, 진행 상태처럼 즉시성이 핵심일 때
- Polling 비용이 높은 경우
- 다자간 동기 상태 갱신이 필요한 화면에서 사용

## 기본 데이터 흐름
1. 클라이언트가 연결 수립
2. 인증/세션 검증 후 구독 채널 등록
3. 서버가 이벤트 발생 시 브로드캐스트
4. 클라이언트는 수신 이벤트로 UI 즉시 반영

## NestJS + React 예시 구상
- 시나리오: 스터디 진행률 공동 편집 동기화
- NestJS: WebSocket Gateway에서 `progress.updated` 이벤트 브로드캐스트
- React: 커스텀 hook이 소켓 이벤트를 구독해 진행 막대 자동 갱신
- 실패 시 재연결 정책과 낙관적 UI 동기화 고려

## 스터디 범위
- 연결 생명주기(생성/재연결/종료) 제어
- 인증 토큰 갱신과 채널 권한 처리
- 이벤트 스키마 통일(버전/타입)
- 실시간 화면(진행률/알림) 구현
