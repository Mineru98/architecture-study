# Outbox Pattern

## 개념
Outbox 패턴은 데이터 저장 트랜잭션과 메시지 발행을 한 트랜잭션 경계로 묶는 방법이다.
- DB 변경과 메시지 발행 불일치 방지
- 이벤트 신뢰성 향상을 위한 패턴

## 언제 쓰는지
- 분산 트랜잭션 없이 일관성 보장이 필요할 때
- DB 변경 후 이벤트 전파 누락을 방지해야 할 때
- 통합 서비스 사이에서 재처리와 추적이 중요할 때

## 기본 데이터 흐름
1. 비즈니스 트랜잭션이 DB와 Outbox 테이블에 동시에 기록
2. Poller/Relay가 Outbox를 읽어 브로커로 전달
3. 메시지 발송 실패 시 재시도
4. 소비자는 멱등하게 처리 후 중복 방지

## NestJS + React 예시 구상
- 시나리오: 과제 제출 후 포인트 적립 이벤트 발행 보장
- NestJS: 과제 상태 변경과 이벤트 레코드 삽입을 같은 DB 트랜잭션에서 수행
- 메시지 큐 리레이어가 `points.earned` 이벤트를 발송
- React: 사용자 포인트 변경 알림을 이벤트/조회 API로 확인

## 스터디 범위
- 트랜잭션 경계와 Outbox 스키마
- Polling 주기와 중복/유실 방지 전략
- 장애 시 재처리 및 DLQ 연계
- 백엔드 단일 트랜잭션에서 메시지 신뢰성 증명
