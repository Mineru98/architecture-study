# MQTT 패턴

## 개념
MQTT는 경량 메시징 프로토콜로 발행/구독 방식에 가깝고, IoT·저대역 폭넓은 환경에 맞다.
- 브로커 기반 메시지 전달
- 토픽 계층 및 QoS 제어

## 언제 쓰는지
- 느린 네트워크, 제한된 대역폭 환경이 있을 때
- 기기 다수에서 경량 실시간 상태를 전송해야 할 때
- 메시지 전달 신뢰성(QoS) 조절이 필요한 경우

## 기본 데이터 흐름
1. Publisher가 토픽에 메시지 발행
2. Broker가 구독자에게 전달
3. Subscriber는 토픽 필터에 따라 수신
4. 처리 결과를 저장하거나 UI 반영

## NestJS + React 예시 구상
- 시나리오: 실습실 출석 센서 상태 수신
- NestJS: MQTT 클라이언트 모듈로 장비 이벤트 수신 후 이벤트 저장
- React: 출석 상태 변화 패널에서 실시간 토픽을 구독해 표시
- 상태가 변하면 서버는 API로 보조 조회 가능 여부 확인

## 스터디 범위
- 토픽 설계 규칙
- QoS와 영속/비영속 메시지 차이
- 스키마(버전)와 보안 인증
- 최소 2개 토픽(상태/경고)로 동작 실험
