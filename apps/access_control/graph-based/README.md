# Graph-Based Structure

## 개요

Graph-Based 구조는 역할, 사용자, 리소스, 그룹을 그래프로 연결하고 경로를 따라 접근 가능 여부를 계산하는 구조다.

## 쇼핑몰 예시

다음 같은 연결을 함께 모델링할 수 있다.

- 사용자 -> 판매자 조직
- 판매자 조직 -> 스토어
- 스토어 -> 브랜드
- 브랜드 -> 상품
- 상품 -> 주문 항목
- 운영자 -> 담당 카테고리

이 구조를 쓰면 "카테고리 운영자이면서 해당 브랜드 담당 조직에 속한 사람만 프로모션 승인 가능" 같은 규칙을 표현하기 쉽다.

## 핵심 개념

- 노드와 엣지로 권한 관계를 표현한다.
- 다대다 관계와 상속, 포함 관계를 함께 모델링하기 쉽다.
- ReBAC와 자주 결합되지만 구조 자체는 별도로 이해할 수 있다.

## React + NestJS 스터디 예시

프론트엔드는 판매자 조직, 브랜드, 카테고리 담당 관계를 시각화한 관리자 패널을 제공한다.
백엔드는 관계 그래프 조회 API와 권한 판정 서비스를 분리한다.

## 적합한 사례

- 입점사, 브랜드, 카테고리 조직이 복잡한 대형 마켓플레이스
- 운영 조직과 판매자 조직이 여러 층으로 연결된 커머스

## 주의점

- 저장소 선택과 조회 성능이 중요하다.
- 순환 관계와 탐색 깊이 제한을 설계해야 한다.
