# Delegation Structure

## 개요

Delegation 구조는 특정 사용자가 가진 권한 일부를 다른 사용자에게 임시 또는 제한적으로 위임하는 구조다.

## 쇼핑몰 예시

- 판매자 대표가 휴가 기간 동안 `seller-staff`에게 상품 수정과 주문 확인 권한만 위임한다.
- 고객센터 팀장이 특정 상담원에게 고액 환불 1차 승인 권한을 하루 동안 위임한다.
- 정산 담당자가 마감 기간에만 외부 회계 담당자에게 정산 조회 권한을 위임한다.

## 핵심 개념

- 위임자와 수임자가 분리된다.
- 만료 시간, 범위, 승인 규칙이 중요하다.
- 운영 대행과 임시 업무 분산에 적합하다.

## React + NestJS 스터디 예시

프론트엔드는 권한 위임 생성 폼과 위임 이력 화면을 제공한다.
백엔드는 위임 테이블을 검증해서 원권한과 위임 범위를 함께 평가한다.

## 필요한 제한 조건

- 위임 가능한 액션을 제한한다.
- 리소스 범위를 제한한다.
- 시작/만료 시간을 강제한다.
- 위임 이력과 실제 사용 로그를 남긴다.

## 적합한 사례

- 판매자 부재 시 운영 대행
- 고객 지원 대리 처리
- 정산 시즌 임시 권한 부여

## 주의점

- 감사 로그가 필수다.
- `refund.approve`, `role.manage` 같은 민감 권한은 재위임 금지 규칙이 필요하다.
