# Flat Structure

## 개요

Flat 구조는 역할 또는 권한이 동일한 레벨에서 독립적으로 정의되는 구조다.

## 쇼핑몰 예시

역할을 다음처럼 서로 독립적으로 둔다.

- `customer`
- `seller`
- `cs-agent`
- `settlement-manager`
- `platform-admin`

각 역할은 상속 없이 필요한 권한을 직접 가진다.

## 핵심 개념

- 상속이나 위임이 없다.
- 각 역할이 직접 권한 집합을 가진다.
- 초기 구현이 단순하다.

## React + NestJS 스터디 예시

프론트엔드는 `seller`, `cs-agent`, `platform-admin` 같은 역할 배열을 받아 메뉴 노출을 제어한다.
백엔드는 역할별 권한 목록을 고정 테이블이나 상수로 관리한다.

## 적합한 사례

- 작은 쇼핑몰 관리자 도구
- 판매자 수와 권한 종류가 많지 않은 커머스

## 주의점

- `order.read` 같은 공통 권한 중복이 쉽게 생긴다.
- 운영 역할이 늘어나면 모든 역할 정의를 일일이 수정해야 한다.
