# Reactive 패턴

## 개념
Reactive 패턴은 데이터/이벤트 스트림을 중심으로 상태 변화를 처리한다.
- Observable/Subject로 비동기 흐름을 선언적으로 조합
- 상태는 이벤트 기반으로 갱신되어 시간 의존성을 관리

## 언제 쓰는지
- 입력/검색 자동완성, 타이머, 롱 폴링처럼 비동기 변화가 빈번할 때
- 상태 의존성이 이벤트 흐름으로 표현되는 도메인일 때
- 취소/재시도/디바운스 로직이 핵심일 때

## 기본 데이터 흐름
1. 사용자 동작 또는 외부 이벤트를 스트림으로 수집
2. filter/map/merge/switchMap 등으로 파이프라인 구성
3. API 응답 또는 소켓 이벤트를 상태 스트림에 반영
4. UI 구독자가 최신 상태 구독 및 렌더링

## NestJS + React 예시 구상
- 시나리오: 실시간 검색 + 추천 리스트 갱신
- NestJS: 검색 API와 추천 캐시 업데이트 이벤트 제공
- React: RxJS 기반 hook(`searchStream$`)에서 디바운스/취소를 처리해 네트워크 호출 최적화
- `BehaviorSubject`로 결과 목록 상태와 로딩 상태를 공유

## 스터디 범위
- RxJS 연산자 선택 기준 정리
- 취소 가능한 요청 흐름 처리
- 스트림 기반 실패 처리와 재시도 정책
- 소규모 이벤트 중심 기능 하나를 스트림으로 풀어보기
