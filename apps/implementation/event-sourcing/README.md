# Event Sourcing

## 개념
Event Sourcing은 현재 상태를 저장하지 않고 상태 변경 이벤트를 누적해 재구성한다.
- 영속 데이터는 이벤트 로그 중심
- 상태는 이벤트 재생(replay)으로 생성

## 언제 쓰는지
- 감사 추적이 강하게 필요한 도메인
- 과거 상태의 재현 및 회귀 분석이 중요한 경우
- 복잡한 동시성 변경 이력 관리가 필요한 경우

## 기본 데이터 흐름
1. 동작이 발생하면 이벤트를 기록(append-only)
2. 이벤트에 버전/메타데이터를 함께 저장
3. 읽을 때 이벤트를 순차 재생해 모델 재구성
4. 조회 성능이 필요하면 별도 Read 모델을 만든다

## NestJS + React 예시 구상
- 시나리오: 학습 로그 감사 시스템
- NestJS: `StudySessionEvents` 테이블/컬렉션에 이벤트 저장, `SessionStateRebuilder`로 상태 복원
- React: 히스토리 타임라인 화면에서 이벤트 목록과 현재 상태를 비교 표시
- 상태 조회는 캐시된 뷰 모델을 사용해 사용자 반응 속도 보완

## 스터디 범위
- 이벤트 스키마 설계와 버전 관리
- 리빌드 성능 이슈와 스냅샷 전략
- 일시적 불일치 처리(동시성, 멱등성)
- 감사 로그 UI를 포함한 소규모 엔드투엔드 시나리오
