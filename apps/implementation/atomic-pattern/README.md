# Atomic 패턴

## 개념
Atomic 패턴은 UI를 가장 작은 재사용 단위(원자)로 나누고, 조합 가능한 블록으로 제품을 구성한다.
- Atom, Molecule, Organism 형태로 조합 단계를 정의
- 상태/스타일/의존성을 최소 단위에서 명확히 분리

## 언제 쓰는지
- 디자인 시스템 정합성이 중요하고 UI 재사용이 빈번할 때
- 공통 컴포넌트와 도메인 컴포넌트를 명확히 분리하고 싶을 때
- 유지보수 비용을 낮추기 위해 컴포넌트 생애주기를 정형화해야 할 때

## 기본 데이터 흐름
1. 백엔드 API에서 raw data 수신
2. 도메인 조합 컴포넌트가 원자 컴포넌트로 데이터 분해
3. 각 원자/조직 컴포넌트가 props로 상태 표현
4. 상위 화면에서 조합해 최종 UI 완성

## NestJS + React 예시 구상
- 시나리오: 리뷰 카드 시스템
- NestJS: 카드, 태그, 작성자 정보를 제공하는 API
- React: `ReviewCard`, `TagPill`, `RatingStars`와 같은 원자 단위 컴포넌트 작성
- API 응답을 조합해 `ReviewBoard`가 여러 원자를 정렬/그리드 구성

## 스터디 범위
- 컴포넌트 분해 기준과 props 계약 설계
- 디자인 토큰/테마 연동 방법
- 공통 컴포넌트 확장(variant, size, 상태) 패턴
- 한 화면에서 3단계 조합 과정을 문서화하여 리뷰
